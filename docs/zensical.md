# Zensical documentation model

This repository documentation follows a **ZENSICAL** model:

- **Z**ero-friction onboarding
- **E**xplicit runtime and operational behavior
- **N**avigable, task-focused structure
- **S**ecurity-first guidance
- **I**mplementation-aware architecture notes
- **C**onsistent terminology and naming
- **A**ctionable runbooks and checklists
- **L**iving docs updated with code and workflows

## Applied in this repository

- Core architecture and lifecycle are documented in dedicated pages.
- Windows service and security guidance are operationally focused.
- Build/test/release instructions match CI workflows.
